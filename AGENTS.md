# Azarmehr Sanat Monorepo — Agent Instructions

## پروژه
آذرمهر صنعت — سامانه یکپارچه مدیریت فروش و CRM (Node.js + Express/Supabase/React)

## زبان و تاریخ
- زبان خروجی و کامنت‌ها: **فارسی** (مگر اینکه کاربر انگلیسی بخواهد)
- تاریخ‌ها: **هجری شمسی** (۱۴۰۵)
- برند: **گروه محصولات غذایی عقرب**

## زیرپروژه‌ها

| سرویس | مسیر | توضیح |
|-------|------|-------|
| Backend API | `backend/` | Express API روی Vercel |
| Admin Panel | `admin-panel/` | پنل مدیریت داخلی (React standalone) |
| Wholesale Portal | `wholesale-portal/` | پورتال مشتریان عمده‌فروش |
| WhatsApp API | `whatsapp-broadcast-api/` | ربات واتساپ (UltraMsg → Vercel → Supabase) |
| Messenger App | `messenger-app/` | پیام‌رسان داخلی |
| Supabase | `supabase/` | کانفیگ و مایگریشن‌های دیتابیس |

## قواعد معماری

1. **منبع داده**: Supabase منبع اصلی است — هیچ JSON file به‌عنوان data source مجاز نیست.
2. **فروش خرد**: فقط از طریق وب‌سایت شرکت (scorpiongroup.ir) — CRM تحلیلی است، نه فروش مستقیم.
3. **فروش عمده**: مسیر مستقل از طریق Wholesale Portal.
4. **احراز هویت**: JWT با bcrypt — دو مسیر جداگانه (admin و customer).
5. **نام محصولات (SSOT)**: `محصولات.txt` در ریشه پروژه، مرجع قطعی نام تمام محصولات است. هر تغییری فقط در این فایل اعمال شود.
6. **قیمت در واتساپ**: ارسال قیمت عمده در واتساپ ممنوع — مشتریان به پورتال هدایت شوند.

## قانون گارانتی و بازگشت وجه (منبع اصلی — SSOT)

سناریوی کامل ۹ مرحله‌ای در فایل `docs/warranty-scenario.txt` ذخیره شده است. این خلاصه برای مرجع سریع است.

**محصولات مشمول (فقط این دو):**
- چای شکسته زرین ۵۰۰ گرمی عقرب
- چای باروتی زرین ۵۰۰ گرمی عقرب
- روی جعبه: عبارت «ضمانت بازگشت وجه» + پیام برند «همه خریداران ۷ گرم چای مهمان عقرب هستند»

**منطق گارانتی:**
- داخل جعبه: ساشه نمونه ۷ گرمی رایگان (به وزن اضافه شده، هزینه ندارد)
- مشتری ابتدا فقط نمونه را مصرف کند
- شرط: پاکت اصلی باز نشده باشد
- اگر پاکت باز شده → مشمول گارانتی نیست

**سناریوی ۹ مرحله‌ای (خلاصه):**
| مرحله | اقدام |
|---|---|
| ۱ | مشتری ناراضی → بپرس: فقط نمونه استفاده شده؟ پاکت باز نشده؟ چگونه دم کردی؟ |
| ۲ | اگر پاکت باز شده → توضیح محترمانه عدم شمول + راهنمایی دم‌آوری |
| ۳ | اگر واجد شرایط → بررسی روش دم‌آوری (آب، زمان، دما) |
| ۴ | ارائه راهنمایی دم‌آوری صحیح |
| ۵ | پیشنهاد امتحان مجدد با روش درست |
| ۶ | اگر منصرف شد → پرونده بسته با وضعیت resolved_by_guidance |
| ۷ | اگر اصرار داشت → نام، شماره، آدرس، کدپستی، محصول، علت را بگیر |
| ۸ | ثبت در پنل ادمین + پیام تأیید |
| ۹ | در همه مراحل: «همه خریداران ۷ گرم چای مهمان عقرب هستند» |

**ذخیره‌سازی خودکار:**
- REFUND_REQUEST و DISSATISFACTION → رکورد در جدول warranty_returns
- وضعیت اولیه: pending
- فیلدها: customer_phone, customer_name, customer_address, product_code, reason

## قوانین مسیریابی فروش (واتساپ)
1. **مسیر خرده و عمده کاملاً جدا هستند** — آدرس پرتال یکی است ولی سبک فروش و لاگین متفاوت
   - خرده: شماره تماس → پرتال فروشگاه (فقط بسته‌بندی، اول سلیقه‌شناسی)
   - عمده: نام کاربری → پرتال فروشگاه (فله + بسته‌بندی، حرفه‌ای و مستقیم)
   - هرگز به مشتری نگو "فعلاً از یه پرتال" — مشتری نباید حس کند مسیرها قاطی هستند
2. **سلام**: هر وقت مشتری کلمه "سلام" فرستاد، اولین جمله پاسخ حتماً با سلام شروع شود (نه فقط اولین پیام)

## سناریوهای فروش واتساپ (SSOT)
نسخه یکپارچه سناریوها در فایل‌های زیر ذخیره شده:
- `docs/Basteh.txt` — سناریوی چای‌های بسته‌بندی (خرده): تشخیص سلیقه → معرفی محصول → CTA فروشگاه آنلاین
- `docs/fale.txt` — سناریوی چای فله و خرید عمده: تشخیص نوع مصرف → معرفی محصول → CTA پخش عقرب

### قوانین ثابت سناریوها:
1. **لینک یا ارجاع نباید قبل از معرفی محصول بیاید** — همیشه اول محصول منطبق معرفی شود
2. **هر بار فقط یک محصول یا یک گروه محصول معرفی شود** — لیست کامل ممنوع
3. **در سناریوی خرده:** اول سلیقه تشخیص داده شود → یک محصول منطبق معرفی شود → CTA به فروشگاه آنلاین
4. **در سناریوی عمده:** اول نوع مصرف تشخیص داده شود → محصول(های) منطبق معرفی شود → CTA به پخش عقرب
5. **CTA نهایی خرده:** "برای مشاهده جزئیات بیشتر و ثبت سفارش، می‌توانید به فروشگاه آنلاین عقرب مراجعه بفرمایید."
6. **CTA نهایی عمده:** "برای ادامه بررسی، مشاهده جزئیات بیشتر و ثبت درخواست خرید عمده، لطفاً از طریق پخش عقرب اقدام بفرمایید."

## استانداردهای کد
- **Type**: commonjs (require/module.exports)
- **Backend**: Express + Vercel Serverless Functions
- **Frontend**: React با Babel standalone (jsx)
- **Database**: PostgreSQL (Supabase)
- **WhatsApp**: UltraMsg webhook → Intent Engine (فارسی) → Supabase
